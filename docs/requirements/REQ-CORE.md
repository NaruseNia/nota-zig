# REQ-CORE: Lexer コア機能

## REQ-CORE-001: comptime 関数 API による Lexer 型生成

- **優先度**: Must
- **ステータス**: Draft
- **説明**: ユーザーが enum とパターン定義を comptime 関数に渡すことで、最適化された Lexer 型がコンパイル時に生成される。Zig のマクロ不在を comptime 関数で代替する設計。
- **受入条件**:
  - [ ] `nota.Lexer(TokenEnum)` 形式で Lexer 型を生成できる
  - [ ] 生成された型が `init()` と `next()` メソッドを持つ
  - [ ] comptime エラーで不正な定義を検出し、分かりやすいエラーメッセージを出す
  - [ ] 生成された型がランタイムで正しくトークナイズを行う
- **依存要件**: なし

## REQ-CORE-002: 文字列リテラルによるトークン定義

- **優先度**: Must
- **ステータス**: Draft
- **説明**: キーワードや演算子などの固定文字列をリテラルで定義できる。内部的には Trie 等で効率的にマッチする。
- **受入条件**:
  - [ ] 固定文字列（`"if"`, `"else"`, `"++"` 等）でトークンを定義できる
  - [ ] 複数のリテラルが共通プレフィクスを持つ場合も正しくマッチする
  - [ ] UTF-8 バイト列としてのリテラルマッチが動作する
- **依存要件**: REQ-CORE-001

## REQ-CORE-003: コールバックによるカスタムマッチ

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 正規表現で表現困難なパターン（ネストコメント、文字列エスケープ等）をユーザー関数でマッチできる。logos の `#[token(callback = ...)]` に相当。
- **受入条件**:
  - [ ] ユーザー定義関数をトークンパターンとして登録できる
  - [ ] コールバック関数は入力スライスと現在位置を受け取り、マッチ長を返す
  - [ ] コールバックが `null` を返した場合はマッチ失敗として扱う
  - [ ] コールバックと正規表現/リテラルの混在が可能
- **依存要件**: REQ-CORE-001

## REQ-CORE-004: 定義順による優先度解決

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 同じ入力に複数パターンがマッチする場合、先に定義されたルールが優先する。ユーザーが直感的に優先度を制御できる。
- **受入条件**:
  - [ ] 同一長のマッチが複数ある場合、先に定義されたトークンが返される
  - [ ] キーワード（リテラル）を識別子（正規表現）より先に定義すれば、キーワードが優先される
  - [ ] 最長一致の中での定義順優先が正しく動作する
- **依存要件**: REQ-CORE-001, REQ-CORE-002

## REQ-CORE-005: `[]const u8` 入力

- **優先度**: Must
- **ステータス**: Draft
- **説明**: Lexer はバイトスライスのみを入力として受け取る。ファイル全体をメモリに読み込んでから渡す想定。
- **受入条件**:
  - [ ] `Lexer.init(input: []const u8)` で初期化できる
  - [ ] 入力のコピーは行わない（ゼロコピー）
  - [ ] 空入力に対して即座に EOF を返す
- **依存要件**: REQ-CORE-001

## REQ-CORE-006: Token 出力形式（slice / offset 両対応）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: トークンの出力形式として、入力スライスへの参照版とバイトオフセット版の両方を提供する。comptime 設定で切り替え可能。
- **受入条件**:
  - [ ] `Token{ .tag, .slice }` 形式（スライス参照）が利用可能
  - [ ] `Token{ .tag, .start, .end }` 形式（オフセット）が利用可能
  - [ ] comptime パラメータで切り替えられる
  - [ ] オフセット版は `u32` を使用し、コンパクトなメモリフットプリント
- **依存要件**: REQ-CORE-001

## REQ-CORE-007: エラートークン方式

- **優先度**: Must
- **ステータス**: Draft
- **説明**: マッチしない入力に遭遇した場合、`.invalid` トークンを返して 1 バイト進める。ストリームが停止せず、パーサー側でエラー報告可能。
- **受入条件**:
  - [ ] マッチしない文字に対して `.invalid` トークンが返される
  - [ ] `.invalid` トークン後も Lexer が正常に動作を継続する
  - [ ] 連続するマッチ不能文字が個別の `.invalid` トークンとして返される
- **依存要件**: REQ-CORE-001

## REQ-CORE-008: Iterator インターフェース

- **優先度**: Must
- **ステータス**: Draft
- **説明**: Lexer は Zig の慣例に従い Iterator パターンで実装する。`next()` が `?Token` を返し、`null` で EOF を表す。
- **受入条件**:
  - [ ] `while (lexer.next()) |token|` パターンで使用できる
  - [ ] EOF で `null` を返す
  - [ ] `reset()` で先頭に戻れる
  - [ ] 現在位置（バイトオフセット）を取得できる
- **依存要件**: REQ-CORE-001
