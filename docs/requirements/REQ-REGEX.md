# REQ-REGEX: 正規表現エンジン

## REQ-REGEX-001: 正規表現パーサー（comptime）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 正規表現文字列を comptime で構文解析し、AST に変換する。不正な正規表現に対してコンパイル時エラーを出す。
- **受入条件**:
  - [ ] 正規表現文字列を comptime で AST にパースできる
  - [ ] 不正な正規表現に対してコンパイルエラーが発生する
  - [ ] エラーメッセージに不正箇所の位置情報が含まれる
  - [ ] サポートする構文: 文字クラス, 量指定子, 選択, グループ, 先読み, 非貪欲
- **依存要件**: なし

## REQ-REGEX-002: NFA 構築（comptime）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: 正規表現 AST から Thompson's construction で NFA を comptime で構築する。
- **受入条件**:
  - [ ] AST から NFA を comptime で構築できる
  - [ ] ε遷移が正しく処理される
  - [ ] 複数の正規表現を単一 NFA に統合できる（各終了状態がトークン種別を識別）
- **依存要件**: REQ-REGEX-001

## REQ-REGEX-003: DFA 変換（comptime）

- **優先度**: Must
- **ステータス**: Draft
- **説明**: NFA を部分集合構成法で DFA に変換し、ランタイム用の状態遷移テーブルを生成する。DFA の最小化も行う。
- **受入条件**:
  - [ ] NFA から DFA への変換が comptime で完了する
  - [ ] 生成される DFA が最小化されている
  - [ ] 状態遷移テーブルがコンパクトに生成される
  - [ ] コンパイル時間が実用的な範囲に収まる（100トークン定義で数秒以内）
- **依存要件**: REQ-REGEX-002

## REQ-REGEX-004: 文字クラスサポート

- **優先度**: Must
- **ステータス**: Draft
- **説明**: `[a-z]`, `[^0-9]`, `\w`, `\d`, `\s` 等の文字クラスをサポートする。ASCII 範囲のみ。
- **受入条件**:
  - [ ] 範囲指定 `[a-z]`, `[A-Z0-9]` が動作する
  - [ ] 否定 `[^...]` が動作する
  - [ ] ショートハンド `\w`(=[a-zA-Z0-9_]), `\d`(=[0-9]), `\s`(=[\t\n\r\f ]) が動作する
  - [ ] 大文字版 `\W`, `\D`, `\S` が動作する
  - [ ] `.` が改行以外の任意のバイトにマッチする
- **依存要件**: REQ-REGEX-001

## REQ-REGEX-005: 量指定子サポート

- **優先度**: Must
- **ステータス**: Draft
- **説明**: `*`, `+`, `?`, `{n}`, `{n,}`, `{n,m}` の量指定子をサポートする。
- **受入条件**:
  - [ ] `*`（0回以上）が動作する
  - [ ] `+`（1回以上）が動作する
  - [ ] `?`（0回または1回）が動作する
  - [ ] `{n}`（正確にn回）が動作する
  - [ ] `{n,}`（n回以上）が動作する
  - [ ] `{n,m}`（n回以上m回以下）が動作する
- **依存要件**: REQ-REGEX-001

## REQ-REGEX-006: 選択・グループサポート

- **優先度**: Must
- **ステータス**: Draft
- **説明**: `|` による選択と `()` によるグループ化をサポートする。キャプチャは不要（Lexer 用途）。
- **受入条件**:
  - [ ] `a|b` が a または b にマッチする
  - [ ] `(ab)+` がグループ単位の繰り返しとして動作する
  - [ ] ネストしたグループ `((a|b)c)+` が動作する
  - [ ] キャプチャグループ機能は不要（非キャプチャとして扱う）
- **依存要件**: REQ-REGEX-001

## REQ-REGEX-007: 先読み・否定先読みサポート

- **優先度**: Should
- **ステータス**: Draft
- **説明**: `(?=...)` による肯定先読みと `(?!...)` による否定先読みをサポートする。DFA ベースでの実装は複雑だが、Lexer 用途で有用。
- **受入条件**:
  - [ ] `(?=...)` がマッチ位置を進めずに先の内容を判定できる
  - [ ] `(?!...)` がマッチ位置を進めずに先の内容を否定判定できる
  - [ ] 先読みと通常パターンの組み合わせが動作する
- **依存要件**: REQ-REGEX-001, REQ-REGEX-003

## REQ-REGEX-008: 非貪欲量指定子サポート

- **優先度**: Should
- **ステータス**: Draft
- **説明**: `*?`, `+?`, `??` の非貪欲（lazy）量指定子をサポートする。
- **受入条件**:
  - [ ] `*?` が最短マッチを返す
  - [ ] `+?` が最短マッチを返す
  - [ ] `??` が最短マッチを返す
  - [ ] 貪欲版と非貪欲版が混在するパターンが正しく動作する
- **依存要件**: REQ-REGEX-005
