# TASK-007: ランタイムエンジン（テーブルドリブン DFA）

- **関連要件**: REQ-PERF-001, REQ-CORE-005, REQ-CORE-007
- **規模**: M
- **依存タスク**: TASK-005

## 実装概要

comptime で生成された遷移テーブルをランタイムで実行する DFA ランナーを実装する。ホットパスの最適化が重要。

## 対象ファイル

- `src/engine/runner.zig`

## 実装ステップ

1. `matchToken` 関数を実装:
   - 開始状態から入力バイトをインデックスに遷移テーブルをルックアップ
   - 受理状態を通過するたびに最後の受理位置を記録（最長一致）
   - デッド状態に到達したら最後の受理を返す
2. コールバックトークンの統合:
   - DFA マッチ前にコールバックを試行（または後に試行し長い方を採用）
   - 定義順に基づく優先度判定
3. マッチ不能時の処理: `.invalid` トークンを返して 1 バイト進める
4. テーブルルックアップのブランチレス化を確認
5. 基本的なトークナイズテストを作成
