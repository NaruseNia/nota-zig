# TASK-005: DFA 変換・最小化

- **関連要件**: REQ-REGEX-003
- **規模**: L
- **依存タスク**: TASK-004

## 実装概要

部分集合構成法で NFA → DFA 変換し、Hopcroft のアルゴリズムで最小化する。遷移テーブルを生成する。

## 対象ファイル

- `src/regex/dfa.zig`
- `src/engine/table.zig`

## 実装ステップ

1. 部分集合構成法を実装:
   - NFA 状態集合を DFA 状態にマッピング
   - ε-closure で初期状態を計算
   - 各入力バイト (0-255) に対する遷移先を計算
   - 新しい状態集合が出現したらキューに追加
2. 受理状態の決定: NFA 受理状態を含む DFA 状態 → 定義順で最初のトークン種別
3. Hopcroft のアルゴリズムで DFA 最小化:
   - 受理/非受理で初期分割
   - 遷移先の分割グループが異なる状態を分離
   - 固定点に達するまで反復
4. `table.zig` に遷移テーブル生成:
   - `[state_count][256]StateId` 形式
   - 受理テーブル `[state_count]?TokenType`
   - デッドステートの除去
5. 等価クラスベースの遷移テーブル圧縮（オプショナル最適化）
6. 小規模パターンでの正確性テスト
7. 中規模パターン（20+ トークン）でのコンパイル時間テスト
