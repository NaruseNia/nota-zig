# TASK-009: コンパイラ統合（Token 定義 → DFA テーブル）

- **関連要件**: REQ-CORE-002, REQ-CORE-003, REQ-CORE-004
- **規模**: M
- **依存タスク**: TASK-005, TASK-007

## 実装概要

ユーザーのトークン定義（enum + パターン）を受け取り、統合 DFA テーブルを生成するコンパイラを実装する。

## 対象ファイル

- `src/engine/compiler.zig`

## 実装ステップ

1. TokenEnum からリテラルトークンを抽出:
   - `@"..."` フィールド名をリテラル文字列として認識
   - `patterns` に含まれるフィールドは正規表現として処理
   - `callbacks` に含まれるフィールドはコールバックとして登録
2. リテラルを正規表現に変換（エスケープ処理）
3. 全パターンの NFA を構築 → 統合 → DFA 変換
4. 定義順（enum フィールド順）を保持し、優先度情報を DFA に反映
5. コールバックトークンのメタデータをランナーに渡す構造を作成
6. コンパイルエラーの検証:
   - `invalid` フィールドがない場合のエラー
   - `patterns` に存在しないフィールド名がある場合のエラー
   - 不正な正規表現のエラー
7. 統合テスト: 複数パターン種別の混在ケース
